Help & Documentation

Templates Overview

Templates are reusable HTML blocks stored in the database. They can be:

  • System: page or blog structure (head, meta, header close/body open, body close)
  • Content: visible content sections (hero, menu, section, paragraph, etc.)

Page Templates

  • New pages get all default page templates (non-blog) in the current order.
  • On Edit Page, you can reorder blocks; this order is saved per-page.
  • Each block has “Use default” to either use the template’s content or a page-specific override.

Blog Templates

  • New blog posts get all default blog templates (slugs starting with blog_).
  • On Add and Edit Post, you can reorder blocks and override content per-post.
  • Blog publishing renders post blocks with variables available in content templates.

Variables in Blog Templates

In blog post blocks, you can use:

  • {{ "{{ title }}" }}: post title
  • {{ "{{ content }}" }}: post content (HTML from editor)
  • {{ "{{ excerpt }}" }}: post excerpt
  • {{ "{{ featured_image_url }}" }}: URL for featured image

Blog Index Templates

The Blog Index is generated using these templates (in order):

  1. Base header (blog_base_header by default)
  2. Meta (blog_meta by default)
  3. Header close/body open (blog_header_close by default)
  4. Menu (blog_menu by default)
  5. Index content (blog_index_content by default) — receives variables:
  • {{ "{{ posts }}" }} — HTML list of post items
  • {{ "{{ categories }}" }} — HTML list of categories
  1. Footer (blog_footer by default)
  2. Body close (blog_body_close by default)

Customizing the Blog Index Template Slugs

You can change which template slugs are used for the index via Settings:

  • blog_index_base_header_slug
  • blog_index_meta_slug
  • blog_index_header_close_slug
  • blog_index_menu_slug
  • blog_index_content_slug
  • blog_index_footer_slug
  • blog_index_body_close_slug

If a slug is empty or missing, the default listed above is used. The index content template receives only two variables: {{ "{{ posts }}" }} and {{ "{{ categories }}" }}.

Template Order

  • Templates tab lists all templates; you can reorder default order.
  • Reordering templates changes defaults. You may optionally reorder existing pages’ blocks to match.
  • Page and blog editors also allow per-item ordering that overrides defaults.

Overrides and Defaults

  • “Use default” means the block uses content from the template record.
  • Unchecking “Use default” enables a custom override stored for that page/post only.
  • Switching back to default restores the template content.

Import/Export

  • Pages and Templates can be exported/imported as JSON.
  • Export supports selecting specific items; Import supports overwrite.

Media

  • Upload images; small/medium/large variants are created based on settings.
  • Hover an image to copy a ready-made <img> tag for any size.

Tip: All admin UI is Bootstrap-based; public output depends entirely on your templates.
